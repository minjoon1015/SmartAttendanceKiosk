주요 프로젝트 (Major Project)

커뮤니티 스타일 LLM 번역 서비스

문제 정의

대부분의 번역 서비스나 LLM은 표준 언어를 기준으로 동작합니다. 그러나 비표준적이고 역동적인 온라인 커뮤니티 환경에서는 단순 번역으로는 뉘앙스나 문화적 맥락을 가진 번역, 밈, 말투 등 이해하기 힘들며 직접적으로 글 작성 시 딱딱하거나 어색한 느낌으로 타사용자들의 공감을 얻기 힘들 수 있습니다.

아키텍처/설계

전체 시스템은 프론트엔드 (React), 백엔드 (Flask), 데이터 및 LLM 엔진 (Ollama)의 세 가지 주요 계층으로 구성하였습니다.

핵심 기능

  • 페르소나 스타일 변환기: 사용자의 한국어/표준 영어 입력 텍스트를 미리 지정된 특정 Reddit 유저의 어투와 문체로 변환하여 작성
  • 목표 지향적 전략분석: 사용자가 글의 목적(도움/공감 받기 등)을 명시하면 인기 게시글 데이터를 분석하여 최적의 길이, 글 구조 등 전략적 조언

나의 역할 및 기여

아키텍처 초기 세팅 및 설계, Reddit API 연동 실시간 데이터 수집 및 업데이트, 코드 리뷰 및 Git/GitHub 배포 관리

결과 및 성과

  • 벡터 임베딩 기능을 추가하여 의미적 유사도 기반 검색 정확도를 향상시킴.
  • 단순 번역이 아닌 목표 지향적이며 문맥 기반 밈 해설 기능을 제공함.

회고 (배운 점 \& 개선점)

  • 배운 점: 페르소나 샘플 전체를 주입하는 것만으로도 특정 스타일을 모방하도록 유도할 수 있다는 점이 AI 기능이 많이 발전했다고 느꼈습니다.
  • 개선점: 멀티모달 통합을 통해 밈 이미지나 스크린 샷을 올리면 텍스트와 이미지를 동시에 분석할 수 있는 기능을 추가하면 좋을 거 같습니다.
